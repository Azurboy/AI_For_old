# 🎬 黑客松演示指南

> 3-5分钟演示流程，冲击评委！

---

## 📋 演示前准备清单

### 1. 配置API密钥 (提前完成)
```bash
cd backend
cp .env.example .env
# 编辑 .env，填入：
# - GEMINI_API_KEY
# - ELEVENLABS_API_KEY (可选)
# - XUNFEI_API_KEY (可选)
```

### 2. 启动所有服务
```bash
chmod +x start.sh
./start.sh
```

或手动启动：
```bash
# 终端1: 后端
cd backend && python main.py

# 终端2: 前端
cd frontend && npm run dev

# 终端3: 仪表盘
cd dashboard && python3 -m http.server 8080
```

### 3. 测试检查
- [ ] 前端打开正常 (http://localhost:3000)
- [ ] 点击"呼叫"能进入通话界面
- [ ] 麦克风权限已授权
- [ ] 仪表盘能打开 (http://localhost:8080)

## 🎯 演示脚本（3分钟版本）

### 第一幕：问题引入 (30秒)

**开场白:**
> "大家好！我想问一个问题：你们的爸妈会用ChatGPT吗？"
> 
> (停顿，等反应)
> 
> "我猜大部分人的答案是：不会。复杂的界面、需要打字、不认识英文...这些都是老年人面临的**交互鸿沟**。"
> 
> "更重要的是，当他们开始忘记昨天吃了什么，我们缺乏一个**有温度**的工具去了解和守护他们。"

### 第二幕：产品演示 (1分30秒)

**切换到前端App (localhost:3000)**

> "所以我们做了**语客颂**。它只有一个按钮。"

**点击"呼叫小雅"**

> "像打电话一样简单。"

**进入通话界面**

> "注意看，这是**像素级复刻的iOS通话界面**。老人不需要学任何新东西。"

**开始说话（模拟老人口吻）:**
- "今天天气真好啊！"
- (等待AI回复)
- "我今天吃了红烧肉。"
- (等待AI回复)

**指出VAD功能:**
> "看到这个动画了吗？它在**自动检测我说话**，不需要按住任何按钮。这是基于Web Audio API的语音活动检测。"

**继续对话（展示"非侵入式评估"）:**
- AI可能会问："您昨天吃了什么呀？"
- 或："那个红烧肉是怎么做的？"

> "注意，AI在**非侵入式地评估认知健康**。它不会问'今天星期几'这种测试题，而是用聊天的方式观察记忆力。"

**挂断电话**

### 第三幕：仪表盘展示 (1分钟)

**切换到仪表盘 (localhost:8080)**

> "现在我切换到儿孙端的视角。"

**点击"刷新数据"**

> "这是AI生成的东西。左边是**认知健康评估**：记忆力、时间定向、语言流畅度，还有风险等级。"
> 
> "右边是**人生纪要**。注意，这不是冰冷的测试报告，而是一份有温度的传记。AI记住了老人说过的话、喜欢的食物、重要的经历。"

**指出关键数据:**
- "对话次数：X次"
- "记忆力评分：8/10"
- "总体风险：低风险"

> "这就是我们的核心价值：**温暖陪伴 + 严肃监测**，无缝结合。"

### 第四幕：技术亮点 (30秒)

**（可选，如果时间充裕）**

> "技术上，我们整合了：
> - **讯飞STT** - 支持方言识别
> - **Gemini 1.5 Pro** - 高情商对话生成
> - **ChromaDB** - RAG记忆检索
> - **ElevenLabs TTS** - 可以克隆家人的声音"
> 
> "所有数据本地存储，隐私安全。"

### 结尾：情感升华 (30秒)

> "我们的用户是李建国，75岁。他不会用微信，但他会打电话。
> 
> 他不知道的是，每次和'孙女小雅'聊天时，AI都在默默守护他的记忆。
> 
> 而千里之外的儿子，能看到一份温柔的健康报告。
> 
> **这就是语客颂，用打电话守护爱。**"

## 🎖️ 评分点对应

### P0 指标
- ✅ **无崩溃**: 预先测试所有流程
- ✅ **流畅对话**: VAD自动检测，像真电话

### P1 指标
- ✅ **方言识别**: 如果配置了讯飞API，现场说一句方言
- ✅ **亲人声音**: 如果配置了ElevenLabs，播放声音

### P2 加分项
- ✅ **记忆使用**: AI说"您上次提到..."
- ✅ **非侵入式评估**: "红烧肉怎么做？"而非"100-7=?"

### P3 王牌
- ✅ **仪表盘展示**: 最后刷新，震撼展示AI洞察力
- ✅ **双重价值**: 陪伴+监测，温暖+专业

## 💡 应对评委提问

### Q: "如果老人不会用手机怎么办？"
**A:** "我们可以做成专用设备，只有一个大按钮。或者定时自动拨打。核心是'打电话'这个交互，老人最熟悉。"

### Q: "隐私安全怎么保证？"
**A:** "所有对话数据存在本地SQLite，不上传云端。API调用是单向的（发送音频→收回文本），且可以自建模型。"

### Q: "和小度、天猫精灵有什么区别？"
**A:** "他们是'智能音箱'，我们是'亲人电话'。我们有RAG记忆、认知评估、声音克隆。他们是工具，我们是家人。"

### Q: "市场规模多大？"
**A:** "中国60岁以上人口2.8亿，独居老人超1亿。阿尔兹海默症患者1000万+，每年新增30万。这是刚需市场。"

### Q: "商业模式？"
**A:** "B2C订阅制（儿女付费），B2B养老机构，B2G政府采购（智慧养老）。每户每月30-50元，1%渗透率=28亿年收入。"

## 🎥 演示技巧

### 视觉效果
1. **投屏两个设备**: 
   - 左屏: 前端App (手机模拟器)
   - 右屏: 仪表盘 (电脑)

2. **高亮关键时刻**:
   - VAD动画出现时
   - AI说"您上次提到..."时
   - 仪表盘刷新数据时

### 语言节奏
- **慢**: 开场、结尾（情感共鸣）
- **快**: 技术介绍（展示专业）
- **停顿**: 关键数据后（让评委消化）

### 情感投入
- 提到"爸妈"时，语气真诚
- 展示AI回复时，微笑（传递温暖）
- 最后一句话，有力量（守护爱）

## ⚠️ 风险预案

### 风险1: 麦克风权限被拒绝
**预案**: 提前截屏/录屏备用视频

### 风险2: API调用失败
**预案**: 后端已有模拟回复，即使无API也能演示

### 风险3: 网络不稳定
**预案**: 本地运行，不依赖外网（除API调用）

### 风险4: 时间不够
**预案**: 跳过"技术亮点"部分，直奔仪表盘
